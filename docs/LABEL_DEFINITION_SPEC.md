## 標記規則詳細定義

### 標記計算邏輯

以每根 15 分鐘 K 線的收盤價為基準，向後看第 6 根 K 線（1.5 小時後）的收盤價，計算漲跌幅，依閾值決定標記。

```python
# 標記計算虛擬碼

def compute_label(candles, index, window=6, threshold=0.015):

current_close = candles\[index\].close

future_close = candles\[index + window\].close

return_rate = (future_close - current_close) / current_close

if return_rate > threshold: return 'BUY' # +1.5%

if return_rate < -threshold: return 'SELL' # -1.5%

return None # HOLD, 略過不納入訓練
```

### 跨日樣本排除規則

台股交易時段 09:00 ~ 13:30，每日共 18 根 15 分鐘 K 線。預測窗口為 6 根，因此每天最後 6 根 K 線（12:00 ~ 13:30）無法取得完整的未來窗口，必須排除。

| K 線時間  | 窗口結束  | 是否可用 | 備註                           |
| -------- | -------- | -------- | -------- |
| 09:00 ~ 11:30 | 10:30 ~ 13:00 | ✅ 可用 | 完整窗口，正常標記 |
| 11:45 ~ 12:00 | 13:15 ~ 13:30 | ✅ 可用 | 剛好在收盤前完成 |
| 12:15 ~ 13:30 | 13:45 ~ 15:00 | ❌ 排除 | 窗口超出收盤時間，無法取得未來價格 |

每日損失樣本：6 根（佔每日 18 根的 33%），可用樣本為每日 12 根。

### 極端值處理規則

台股設有 ±10% 漲跌停限制，極端值需要特殊處理，避免污染模型學習。

| 情境 | 處理方式 | 理由 |
| -------- | -------- | -------- |
| 漲幅 > +9%（接近漲停） | 排除，不納入訓練 | 漲停板造成流動性消失，不具可交易性 |
| 跌幅 < -9%（接近跌停） | 排除，不納入訓練 | 同上，跌停板無法出場 |
| 成交量為 0 | 排除，不納入訓練 | 無流動性，指標失真 |
| 開盤價 = 收盤價（鎖住） | 標記後保留，加 flag 欄位 | 漲跌停鎖住仍有參考價值，但需標記供後續分析 |

## MVP 規則式邏輯規格

### 目的

XGBoost 模型訓練需時（預計 2026-06 完成），為確保 2026-05-16 整合測試時程不受影響，/predict 端點先以規則式邏輯實作，6 月完成訓練後無縫替換。

### 規則式評分邏輯

基於 Rust 提供的技術指標，透過加權評分決定信號方向與信心分數：

| 條件 | 方向 | 權重 | 邏輯說明 |
| -------- | -------- | -------- | -------- |
| RSI < 30 | BUY | +2 | 超賣區間，反彈機率較高 |
| RSI > 70 | SELL | \-2 | 超買區間，回調機率較高 |
| MA20 > MA50（多頭排列） | BUY  | +1 | 中期趨勢向上 |
| MA20 < MA50（空頭排列） | SELL | \-1 | 中期趨勢向下 |
| MACD Histogram > 0（翻多） | BUY  | +1 | 短期動能轉正 |
| MACD Histogram < 0（翻空） | SELL | \-1 | 短期動能轉負 |
| 收盤價 < Bollinger 下軌 | BUY  | +1 | 價格偏離均值過大，均值回歸預期 |
| 收盤價 > Bollinger 上軌 | SELL | \-1 | 同上，反向邏輯 |

```python
# 信心分數計算（規則式版本）

total_score = sum(weights) # 範圍: -5 ~ +5

up_probability = (total_score + 5) / 10

down_probability = 1 - up_probability

confidence_score = 0.5 # 規則式固定中等信心

model_version = "rules_v1.0"
```

* 切換點設計
  * 規則式邏輯需有明確的 feature flag 切換點（如 MODEL_TYPE=rules / xgboost），6 月 XGBoost 上線後只需修改環境變數，其餘程式碼不需改動。
